Why 1D Register Tiling (Work-Per-Thread)?

Is this 1D Tiling? Yes! Each thread computes a 1D column vector (in our viz, a \(4 \times 1\) tile) of the output block.

The Core Idea: We load a single value from Matrix B in shared memory into a fast, private hardware Register (Btmp).

Increasing Arithmetic Intensity: Because the thread calculates multiple rows (TM), it can reuse Btmp to multiply against multiple elements from Matrix A.

\(\text{SRAM Reads for B} \div \text{TM}\)

The Impact on the "Memory Wall":

Standard Shared Memory tiling fixes the Global Memory bottleneck, but replaces it with a Shared Memory bottleneck (Shared Memory is fast, but Registers are vastly faster).

  • Without Register Tiling (TM=1): 1 Shared Read of A + 1 Shared Read of B = 1 FMA instruction. (Ratio 2:1)
  • With Register Tiling (TM=4): We read Btmp once. Then we read 4 values from A to perform 4 FMA instructions. That's 5 Shared Reads for 4 FMAs. (Ratio 1.25:1)
  • With TM=8: 9 Shared Reads for 8 FMAs. (Ratio ~1.1:1). The GPU is no longer starved for memory and can calculate at maximum theoretical speed!

Why is the Shared Tile for A (8×4) larger than B (4×4)?

It directly relates to the work each thread is doing! Our block has 8 threads arranged in a 2×4 grid. Since each thread computes a 4×1 vertical strip (TM = 4), the block collectively computes an 8×4 block of the output Matrix C.

To compute an 8×4 block of C, the threads need 8 rows from Matrix A but only 4 columns from Matrix B. Thus, the tile fetched from A (As) must be twice as tall as the tile fetched from B (Bs).
