Why two loops?

In a naive kernel, 1 thread calculates 1 output element. It loads 1 value from A and 1 from B for every multiplication.

With Register Tiling, 1 thread calculates TM=4 output elements! To do this efficiently:

  • The dotIdx loop iterates through the K dimension (size BK=4).
  • We load one value from Bs into a fast register (regB).
  • The resultIndex loop multiplies that single regB value against TM=4 different values from As.

Result: We reused the memory loaded into regB 4 times! This drastically reduces memory bandwidth bottlenecks.
